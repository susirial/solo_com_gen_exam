# 工作流说明

本 Skill 的固定执行顺序如下：

1. 确认任务类型
2. 确认关键输入
3. 输出组卷方案
4. 等待用户确认
5. 生成完整试卷内容
6. 校验结构
7. 渲染教师版 / 学生版 HTML

## 1. 任务类型

优先识别属于以下哪类任务：

- 新建整卷
- 调整组卷方案
- 重生单题
- 编辑单题
- 只做结构化整理
- 只做 HTML 渲染

## 2. 关键输入

最少应确认：

- 学段
- 年级
- 学科
- 试卷类型
- 总分
- 时长
- 分节结构
- 知识点范围
- 难度要求
- 输出形式

## 3. 方案阶段

方案阶段不要直接出整卷题目。至少先给出：

- 标题建议
- 分节结构
- 题量和分值配置
- 知识点覆盖
- 难度分布
- 待确认项

## 4. 内容阶段

只有方案已确认后，才生成完整试卷内容。内容阶段应优先整理为统一结构，避免直接把自然语言塞给 HTML 模板。

## 5. 校验与渲染

- 用 `validate_plan.py` 校验方案
- 用 `normalize_paper.py` 整理试卷数据
- 用 `validate_paper.py` 校验完整试卷
- 用 `render_html.py` 渲染教师版 / 学生版 HTML
